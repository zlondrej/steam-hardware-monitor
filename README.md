## Protocol Buffer details

The Steam Hardware Monitor now uses the official **Protocol Buffers library** for type-safe message serialization/deserialization.

- âś… Auto-generated protobuf classes from `.proto` schema
- âś… Official `google.protobuf` library
- âś… Fields extracted automatically
- âś… Type-safe field access
- âś… Full support for nested messages

See `steam_hardware.proto` for the complete schema.

### Generated Code

The `steam_hardware_pb2.py` file is automatically generated by `protoc` (Protocol Buffer Compiler):

```bash
pipenv run python3 -m grpc_tools.protoc -I. --python_out=. steam_hardware.proto
```

This creates:
- `CHardwarePackageDetails` class
- `CHardwareItem` class
- `GetHardwareItemsResponse` class
- `GetHardwareItemsRequest` class
- `HardwareRequestOptions` class
- Auto-serialization/deserialization methods

### Regenerating Code

If the Steam API schema changes:

1. Update `steam_hardware.proto` with new fields
2. Regenerate Python code:
   ```bash
   pipenv run python3 -m grpc_tools.protoc -I. --python_out=. steam_hardware.proto
   ```
3. The changes are automatically available in `steam_hardware_pb2.py`

## Why Protobuf?

1. **Binary Format**: 22 bytes response vs 1MB+ HTML
2. **Type Safety**: Fields are strongly typed
3. **Versioning**: Schema evolution built-in
4. **Performance**: Optimized serialization
5. **Standardized**: Industry standard (Google, Facebook, etc.)
6. **Maintainability**: Schema is documentation
